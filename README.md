# DATA.LABZ
> Presence-first co-working for remote builders — OCTech Services

---

## What this is

DATA.LABZ is a digital co-working space where builders declare what they're working on and lock in alongside others in real time. A task is a room. Presence is visible. The work is the point.

No social feed. No likes. No noise. Just builders, tasks, and accountability.

---

## Before you engage

Read this file and spend a few minutes with the repo before joining or opening a room.

Not a hard rule — a norm. When everyone has context, conversations are sharper, collaboration is faster, and presence here means something. People who skip this step are easy to spot, and it shows.

This is the standard we hold here.

---

## How it works

| Step | Action |
|---|---|
| 1. Lock in | Open a room. The room name IS your task — be specific. |
| 2. Set milestones | Declare what you want to accomplish this session before going active. |
| 3. Show up | Your visibility and presence are live. Others can see you working. |
| 4. Sign off | Close your room when done. Session history and milestones are preserved. |

---

## Room etiquette

- **One active room per builder.** If you're already in one, finish it or pause it.
- **Room name = the actual task.** Not "working", not "online" — what are you actually building?
- **Paused means on a break**, not gone. Close the room if you're done for the day.
- **Read the room before jumping in.** Check the task name and milestones — know the context first.
- **Frozen rooms** are rooms that met their milestone target. Respect the history.

---

## Locations

DATA.LABZ supports physical venues. If you're working from a partner café or co-working space, check in via the venue QR code to surface your location in the hive. Location-gated rooms are reserved for builders physically on-site.
